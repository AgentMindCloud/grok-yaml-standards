# CONTRIBUTING.md

Thank you for helping make **grok-yaml-standards** the official community reference for Grok YAML magic on X!

### How to Contribute

1. Fork the repo  
2. Create a branch: `feat/your-feature-name` (example: `feat/grok-agent-improvement`)  
3. Update the relevant files (`example.yaml`, `schema.md`, `usecases.md`, etc.)  
4. Ensure every change remains **forward-compatible** with `grok-install.yaml`  
5. Open a Pull Request with a clear description of your changes

### We Welcome
- New real-world use cases for X users, devs, and companies  
- Security hardening suggestions  
- Schema improvements and validation rules  
- Better examples, documentation, or translations  

**All contributors** will be credited in the next release notes and on X.

Let’s build the definitive Grok YAML ecosystem together! 🚀
